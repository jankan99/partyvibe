🔧 Supabase Setup Guide

Takes about 10 minutes. Everything is free on Supabase's free tier.

Step 1 — Create project
Go to supabase.com → New Project. Choose a region close to your guests.
Step 2 — Create tables
Go to SQL Editor and run this:
-- Events table
create table events (
  id text primary key,
  party_name text,
  host_name text,
  vibe text,
  welcome text,
  emoji text,
  visual_theme text,
  font_family text,
  active boolean default true,
  purged boolean default false,
  purged_at timestamptz,
  created_at timestamptz default now()
);

-- Guests table
create table guests (
  id text primary key,
  event_id text references events(id),
  name text,
  selfie_url text,
  color text,
  created_at timestamptz default now()
);

-- Quiz answers
create table quiz_answers (
  id text primary key default gen_random_uuid()::text,
  event_id text references events(id),
  guest_id text references guests(id),
  answers jsonb,
  created_at timestamptz default now(),
  unique(event_id, guest_id)
);

-- Photos table
create table photos (
  id text primary key,
  event_id text references events(id),
  hero_name text,
  theme text,
  emoji text,
  photo_url text,
  reactions jsonb default '{"heart":0,"fire":0,"laugh":0,"wow":0}',
  created_at timestamptz default now()
);

-- Reaction increment function
create or replace function increment_reaction(
  photo_id text, reaction_type text
) returns void language plpgsql as $$
begin
  update photos
  set reactions = jsonb_set(
    reactions,
    array[reaction_type],
    to_jsonb((reactions->>reaction_type)::int + 1)
  )
  where id = photo_id;
end;
$$;

-- Enable realtime
alter publication supabase_realtime add table guests;
alter publication supabase_realtime add table photos;
Step 3 — Storage bucket
Go to Storage → New bucket → name it partyvibe → set to Public.
Step 4 — Row Level Security
For testing: in each table → RLS → Disable RLS. For production, add proper policies.
Step 5 — Get your keys
Project Settings → API → copy Project URL and anon public key → paste into the top of this file.
Step 6 — Host it
Deploy on Vercel, Netlify, or Supabase Hosting so guests get a real URL to scan.
